# ARGUS-INT v1.1.0 Roadmap

Cette feuille de route a été construite à partir des premiers retours post-lancement (v1.0.0).

## 🚀 Fonctionnalités Majeures (Features)
- [ ] **Support Kubernetes Étendu** : Fournir des Helm charts complets et des Kubernetes Operators pour déployer ARGUS-INT sur des clusters hybrides.
- [ ] **Nouveaux Modules OSINT** :
  - Intégration native Shodan & Censys (via clés API configurables).
  - Module HaveIBeenPwned (Détection de mots de passe fuités).
- [ ] **Optimisation vLLM** : Implémenter le batching dynamique et le support Multi-GPU (Tensor Parallelism) pour fluidifier les instances avec >10 analystes simultanés.

## 🛠️ Améliorations de l'Interface (UI/UX)
- [ ] **Dark Mode Customisable** : Palettes de couleurs spécifiquement étudiées pour la réduction de la fatigue oculaire lors des gardes de nuit.
- [ ] **Raccourcis Clavier** : Navigation intégrale au clavier dans le graphe d'investigation (Vim-like bindings).

## ⚡ Performance & Scalabilité
- [ ] **Milvus Vector Store** : Refonte de l'indexation (HNSW / IVF_FLAT) pour réduire la consommation RAM de 30% lors du requêtage des embeddings CLIP.
- [ ] **Redis Caching** : Mise en place d'un cache hiérarchique avancé pour limiter la sollicitation des modèles IA sur des requêtes récurrentes.

## 🐛 Corrections de Bugs (Non-Critiques)
- [ ] Corriger le glitch visuel lors du zoom extrême sur un nœud Neo4j dans le frontend.
- [ ] Fixer la rotation des logs Docker qui peut parfois échouer si l'espace disque < 5%.
