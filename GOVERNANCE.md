# ARGUS-INT Governance Model

ARGUS-INT adopte un modèle de gouvernance basé sur l'expertise technique et l'engagement envers la sécurité opérationnelle (OPSEC).

## Les Rôles
1. **Core Maintainers** : L'équipe de fondateurs. Possèdent les clés GPG maîtres. Décision finale sur l'architecture et les releases.
2. **Committers** : Contributeurs réguliers ayant un historique prouvé. Peuvent merger les PR (nécessitent 2 approvals).
3. **Contributors** : Tout développeur soumettant des Pull Requests signées.

## Processus de Décision (RFC)
Toute modification architecturale majeure (ajout d'une base de données, nouveau modèle IA) doit passer par un processus de **Request For Comments (RFC)** dans les GitHub Discussions. Le consensus technique est recherché, mais en cas de désaccord, l'arbitrage final revient aux Core Maintainers, avec la sécurité comme priorité absolue.
